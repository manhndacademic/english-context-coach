# Isolate database types from core domain

We decide to decouple database row types (Drizzle schema types) from the core domain interfaces (Lesson and Memory ports/types). Previously, core domain logic and adapters shared the exact same database-oriented TypeScript types directly imported from `@/db/schema`, which violated layering boundaries and forced domain logic to adapt to database-specific constraints (such as nullable fields). By defining clean domain-specific entities (e.g., `Lesson`, `KeyPhrase`, `Attempt`, `MistakePattern`) in the respective domain contexts and aliasing database schema types as `Db...` inside repository adapters, we protect the core business logic from database schema changes. The trade-off is a slight increase in boilerplate and type casting within repository adapters, but it guarantees type safety, strict architectural layering, and testability without dependencies on the database schema.
